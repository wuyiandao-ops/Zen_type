# ZenType 專案演進與優化

這份文件記錄 ZenType 從最初的雛形到現在的設計歷程,以及一路上做的優化與取捨,
方便理解「為什麼是現在這個架構」。

核心其實是一次架構大轉向:
**從「借 Win+H 的事後校正小工具」→「擁有完整辨識管線的離線語音輸入法」。**

---

## 三個階段

### ① 起點:Win+H 輔助工具(PyQt6 版)
- 靠 Windows 內建 **Win+H** 做語音辨識,使用者把聽寫結果貼進程式的文字框。
- 功能:即時套用「錯誤更正表」+ 用雲端 Gemini 分析使用者的手動修改、自動學專有名詞。
- 資料存在 SQLite(歷史、字典、單字庫)。

### ② 轉折:認清 Win+H 的天花板
- 目標明確化:要做一個能**取代 Typeless** 的輸入法。
- 發現兩個根本限制:
  1. Win+H 是黑箱,**沒辦法餵詞彙進去改善辨識**,只能事後校正。
  2. 抓不到系統級輸入,只能「貼進自己的框再複製」,體驗不連貫。
- 結論:要真的好用,**必須自己擁有辨識引擎** → 改用 SenseVoice(FunASR,即 CapsWriter 的底層引擎)。

### ③ 現在:ZenType(全新自有管線)
```
按住右 Ctrl 錄音 → SenseVoice 離線辨識 → 校正表替換 → Qwen 2.5 潤稿 → 貼到游標
```
離線、保護隱私、繁體中文、免月費。

---

## 關鍵優化(Before → After)

| 面向 | 起點(Win+H 版) | 現在(ZenType) |
|---|---|---|
| 辨識引擎 | Win+H(黑箱、不可控) | SenseVoice 本地引擎(可控、離線) |
| 觸發方式 | 按 Win+H 叫出聽寫 | 按住右 Ctrl 說話、放開送出 |
| 輸出方式 | 貼進框 → 手動複製 | 全域直接貼到任何 App 的游標 |
| AI 角色 | 只做專有名詞校正 | 加去贅字、整句潤稿 |
| 潤稿 AI | 雲端 Gemini(要金鑰、要錢) | 本地 Ollama + Qwen 2.5(離線、免費) |
| 隱私 | 文字送雲端 | 音訊/文字全留本機 |

---

## 具體優化,分六類

**架構**
- 從「事後校正層」升級為「辨識 → 校正 → 潤稿 → 貼字」完整管線。
- 辨識與潤稿都搬到本地(SenseVoice + Ollama),全離線。

**中文正確性**
- 加 opencc 簡轉繁(s2tw):SenseVoice、Qwen 都輸出簡體 → 決定性轉回繁體(台灣)。
- 清掉 Qwen 亂插的空白、SenseVoice 帶出的 emoji。
- 校正表雙軌:專有名詞用純字串替換(零延遲、可靠),不勞煩 AI。

**穩定性**
- 潤稿逾時 fallback:AI 太慢或失敗 → 直接輸出原文,絕不卡住打字。
- 模型預熱 + keep_alive:消除第一次的冷啟動卡頓。
- 背景執行緒處理辨識/潤稿/貼字,不卡住鍵盤熱鍵。

**效能**
- 選用 qwen2.5:3b 小模型,平衡速度與品質,低端機也能跑。

**工程 / 相依修正**
- Gemini SDK 從已棄用的 `google.generativeai` 遷移到新版 `google-genai`。
- 修掉一批 bug:游標跳動、單字重複判斷死碼、背景執行緒管理、單字庫沒回饋 AI。
- 全面防亂碼:主控台 UTF-8、`.bat` 純 ASCII、CSV 用 utf-8-sig。

**散布 / 開源**
- PyInstaller 打包 EXE 支援(`--collect-all` 收齊 opencc 字典與原生 DLL)。
- 一鍵安裝/啟動(`install.bat` / `start.bat`)+ 新手圖文指南,內含辨識服務 → 無經驗者也能裝。
- GitHub 上架(MIT 授權),並清掉公開提交裡的私人資訊。

---

## 一句話總結

> 從「幫 Win+H 擦屁股的校正小工具」,進化成「自己掌握辨識 + 潤稿、離線又保護隱私的繁中語音輸入法」。
> 最核心的一步,是想通了「要好用就得自己擁有辨識引擎」,於是把 Win+H 換成 SenseVoice。
